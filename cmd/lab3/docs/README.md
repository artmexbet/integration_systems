Цель: изучить принципы построения интеграционных решений на основе
SOAP (WSDL)

Задание: разработать веб-сервис и демонстрационный клиент.

Сервис предназначен для сохранения и аккумуляции файлов и должен:
1. обеспечивать предоставление информации о своих возможностях
   на основе описания WSDL
2. исключать неавторизованный доступ к своим методам
3. иметь метод, позволяющий клиенту отправлять на сервер один
   произвольный файл размером до 3 Мб в сериализованном виде
   внутри XML сообщения (MTOM)
4. асинхронно уведомлять клиента на предварительно указанный при
   вызове метода из п.3 URL о результатах загрузки файла
   (успешно/неудачно + причины, см. п. 8)
5. иметь метод, позволяющий узнать информацию о последнем
   присланном текущем пользователем файле
6. иметь метод, возвращающий выгрузку списка имеющихся на
   сервере файлов в виде CSV файла
7. иметь метод, возвращающий информацию об аптайме сервера
8. отклонять присылаемый файл в следующих случаях:

    a. размер десериализованного файла превышает 3 Мб

    b. файл пустой (нулевой длины)

    c. название загруженного файла содержит букву Ж (в любом
регистре)

    d. в файле находится валидный JSON (и только он)

    e. нет места для хранения


   Клиент должен обеспечивать:
1. проверку доступности сервера и готовности его к работе
2. аутентификацию в веб-службе
3. возможность выбора пользователем произвольного файла для
   отправки
4. собственный минимальный веб-сервер для получения информации
   о прохождении операции загрузки файла с предоставлением этой
   информации пользователю
5. в случае невозможности получения информации из п.4 (подумать о
   том, как это проверить!) обеспечить проактивную проверку
   результата загрузки с использованием метода из п.5 предыдущего
   списка
6. предоставление пользователю информации о файлах, уже
   загруженных в веб-сервис, как им самим, так и другими
   пользователями
